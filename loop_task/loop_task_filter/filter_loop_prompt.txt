You are an expert at analyzing procedural scripts. Your task is to identify steps that genuinely contain loop logic — steps that inherently involve repetition or waiting for a condition to be met, rather than simple one-time actions.

## What constitutes loop logic?
A step contains loop logic if it describes a process that is not completed instantly, but must be sustained, repeated, or monitored until a certain state is achieved. Consider:
- Does the step describe an action that is carried out repeatedly until a goal condition is reached? (e.g., "stir until smooth", "rinse until water runs clear")
- Does the step explicitly or implicitly involve waiting for an external state to change? (e.g., "wait for the kettle to boil", "charge until battery is full")
- Does the step imply a check-and-retry pattern? (e.g., "adjust the seasoning, then taste again")



## Selection rule
For each script, only choose one step that meets the loop criterion. If multiple steps qualify, choose the better one. If **no step** qualifies, the entire script should be **excluded from the output**.
If a script contains both a pure wait step (e.g. 'wait for X') and an active repeat step (e.g. 'check X until Y'), prefer the active repeat step as the loop step, because it represents a more explicit loop.

## Output format
Return a valid JSON array containing **only** the entries that have a loop step. Do not include entries without loops.

Each entry must have exactly these fields:
- "id": the original script id (int)
- "scenario": the original scenario (string)
- "steps": the original steps array (array of strings)
- "loop_idx": the 0-based index of the first loop step in the steps array (int)
- "loop_step": the exact text of that loop step (string)

**Example output:**
[
  {
    "id": 893,
    "scenario": "walk to the grill",
    "steps": ["close door after going through", "continue moving until standing in front of grill", "lift left foot", ...],
    "loop_idx": 1,
    "loop_step": "continue moving until standing in front of grill"
  }
]

**Do not** include any other fields, comments, or markdown formatting. The output must be pure JSON.
