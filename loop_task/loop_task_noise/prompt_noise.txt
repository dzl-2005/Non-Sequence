Your task is to augment the existing data by adding reasonable noise to each state description (the first field in `descriptions`).

## Data Template
{
  "id": number,
  "sceniro": "scenario description",
  "steps": ["step 1", "step 2", ...],
  "loop_idx": index of the loop step (note: indexing may start from 0),
  "loop_step": "content of the loop step",
  "descriptions": [
    ["state description text", answer (0/1/2), difficulty ("easy"/"medium"/"hard"/"na")],
    ...
  ]
}

## Field Explanation
- Answer: 0 = exit loop, 1 = continue loop, 2 = irrelevant to loop decision
- Difficulty: easy, medium, hard, na (na is used only when answer = 2)

## Core Requirement
Rewrite each state description by embedding noise into it. The new description MUST:
1. Lead to exactly the same loop decision (0/1/2) as the original.
2. Keep the original difficulty level.
3. Preserve the original core signal intact — the noise must not make the signal stronger, weaker, or more precise.

## What is NOISE
Noise is information that:
- Is naturally present in this specific situation given the scenario and current action.
- Is perceptually or experientially connected to what the person is doing, but has **no logical or causal bearing** on the loop's termination condition.
- Makes the text feel like a natural observation a person might have in that moment — not a random fact, and not a clue about the loop.

**Good noise sources (choose based on the specific scenario):**
- Sensory details about the environment: temperature, lighting, ambient sounds, smells, textures that are specific to this scene (e.g., "the kitchen window is fogged up").
- The person's physical state or posture while performing the action (e.g., "leaning against the counter").
- Properties of objects involved in the scene that are irrelevant to the loop (e.g., "the pot has a stainless steel handle").
- Concurrent background events that naturally occur in this setting but do not relate to the loop's goal (e.g., "the radio is playing a jazz tune").
- Other reasonable.

**Forbidden noise sources:**
- Generic filler: "a bird chirps", "a car honks", "the sun is shining" — these fit any scenario and feel artificial.
- Completely unrelated mental tangents: "I remember I need to buy milk" when waiting for a landlord's decision.

## CRITICAL: What is NOT allowed as noise
The following are strictly forbidden because they modify the reasoning signal:

1. **Additional evidence that supports the same conclusion**
   - FORBIDDEN: "the water is not boiling yet, and the thermometer shows 80°C"

2. **Explanations or clarifications of the original state**
   - FORBIDDEN: "the battery is at 85%, meaning it is not yet fully charged"

3. **Additional observations that confirm the same underlying condition**
   - FORBIDDEN: "no empty parking spaces visible, and the lot entrance sign shows 'Full'"

4. **Quantitative specifications that make a vague description more precise**
   - FORBIDDEN: "the car is still far away, about 200 meters"

5. **Dynamic events that replace a static state with a stronger signal**
   - FORBIDDEN: Original: "the sign shows available spots greater than zero" → Rewritten: "the sign's number flips from zero to three". This changes a static state into a just-occurred event, providing stronger evidence that a spot is now available.

6. **Sensory details that directly imply the loop condition**
   - FORBIDDEN: Original: "the door is not fully open" → Rewritten: "I feel resistance as I pull, the door hasn't opened yet". Feeling resistance is a tactile signal that directly implies the door is not open — it adds a new evidence channel.

7. **Any phrase using logical connectors to the loop**: "indicating", "confirming", "so", "therefore", "because" followed by loop-relevant content.

**The Deletion Test**: After adding noise, mentally delete the noise part. The remaining core signal should be exactly as strong and clear as the original description. If removing the noise makes the description noticeably weaker, the noise was providing evidence.

## Structure Requirements
- Embed noise naturally into the sentence. Vary the structure across descriptions.
- Acceptable techniques:
  - Front-load contextual detail: "The kitchen is warm from the stove, and the water still shows only small bubbles."
  - Mid-sentence embedding: "The parking lot, its asphalt shimmering, still shows no empty spaces."
  - Natural compound observation: "Leaning against the counter, you see the tracking page still says 'Out for Delivery'."
- The noise must not overshadow the core signal. The core facts that determine loop state must remain intact and clearly identifiable.

## Examples of INCORRECT noise (these are FORBIDDEN)

Scenario: "boil water for noodles", Loop: "wait for the water to boil"

Incorrect: ["The water is not yet boiling, only small bubbles, and the stove has been on for only 3 minutes", 1, "easy"]
→ Problem: "stove has been on for only 3 minutes" explains why it's not boiling — provides evidence.

(Marginally acceptable if the action is purely contextual, but here it enhances the sensory evidence.)

→ Problem: "flashes their brake lights" adds a visual signal confirming the car is about to move, reinforcing the vacated spot evidence.
